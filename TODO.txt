Тестовое задание
Часть 1. Базовая функциональность
Задача:
Разработать простое веб-приложение «Список клиентов», состоящее из:
Frontend: Vue 3
Backend: Yii2 (basic)

 Frontend (Vue 3):
Функциональность:
Просмотр списка клиентов.
Создание нового клиента (форма с названием компании и штатом).
Обновление данных клиента (редактирование названия компании и штата).
Удаление клиента.
Возможность изменить статус клиента (Active / Inactive).
Использовать Composition API.
Запросы к серверу — через axios.
Требования:
Отображение клиентов в виде списка или таблицы.
У каждого клиента должны быть поля:
id
name
state
status
Статус клиента можно менять без перезагрузки страницы.

 Backend (Yii2 Basic):
Реализовать REST API с помощью контроллера.
URL-префикс:
/api/client
Методы:
GET /api/client — список всех клиентов.
POST /api/client — создать клиента.
GET /api/client/{id} — получить клиента.
PUT /api/client/{id} — обновить данные клиента.
DELETE /api/client/{id} — удалить клиента.

Требования:
Использовать ActiveRecord модель Client.
Добавить валидацию:
name — обязательное поле.
state — обязательное поле.
Вернуть JSON-ответы.
Авторизация и дизайн не требуются.
Допускаются упрощения — главное корректная логика и аккуратный код.

Контекст:
Нам нужно отслеживать клиентов, у которых давно не было никакой активности.
Задание:

Какие поля вы добавите в модель?
Как определять “неактивных по активности” клиентов?
Как это будет отображаться в интерфейсе?
